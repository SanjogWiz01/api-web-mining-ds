# Expansion Roadmap

This repository is a realistic v1. To grow it into a full book-sized course, expand in staged PRs rather than adding filler.

## Suggested Modules

- API authentication deep dive: API keys, OAuth, JWT, signed URLs, and token refresh.
- Cursor pagination and resumable checkpoints.
- Async crawling with bounded concurrency and per-domain rate limits.
- Browser-based extraction with Playwright for JavaScript-heavy pages.
- Data contracts with Pydantic.
- Larger storage patterns with DuckDB and PostgreSQL.
- Scheduling with cron, GitHub Actions, or Airflow.
- Monitoring for schema drift and failed jobs.
- Practical NLP on collected news, reviews, and comments.
- Responsible data collection, privacy, copyright, and governance.

## Capstone Ideas

- Public weather and air-quality analysis.
- GitHub repository trend mining.
- Product catalog monitoring with price history.
- News collection and sentiment analysis.
- Research paper metadata mining.

Each capstone should include setup, collection, cleaning, storage, analysis, tests, and a report. That is how the course becomes substantial without pretending that page count equals quality.
